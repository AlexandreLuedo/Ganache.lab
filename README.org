* Ganache.lab
Le premier équilibreur de ganache multiplateforme !

#+begin_quote
[!NOTE]
Ceci est mon premier projet, il sert de support à mon apprentissage de Dart & Flutter. L'architecture est donc amenée à évoluer régulièrement.
#+end_quote

** Fonctionnalités
L'application Ganache.lab permet l'équilibrage précis de ganaches via deux approches principales :

*** 1. Génération de recette (Mode principal)
C'est le cœur de l'application. L'utilisateur définit ses contraintes (type d'utilisation, grammage final désiré, etc.). L'application se charge ensuite de générer une recette équilibrée en s'appuyant sur les paramètres définis dans le fichier de configuration.

*** 2. Importation de recette (Roadmap)
/Fonctionnalité non implémentée pour le moment./
L'objectif est de permettre à l'utilisateur de saisir ingrédient par ingrédient sa propre recette pour vérifier son équilibrage et le corriger si nécessaire.

** Architecture Technique

*** Conventions de code
- *Commentaires :* Rédigés majoritairement en **Français**.
- *Variables :* Nommées en *Anglais* (camelCase).
  - *Note :* Certaines variables historiques possèdent encore des noms hybrides (ex: ~numberCadres~). Un nettoyage progressif est en cours pour uniformiser le tout.

*** Structure du projet
- =GanacheApp()= : Point d'entrée de l'application.
- =navigation.dart= : Gestion de la navigation principale ("Home page").
- =privateConfig.yaml= : Fichier de configuration stockant les constantes et variables d'ajustement pour la table analytique.
- =engine.dart= : Moteur de calcul utilisant les données de configuration.

*** Gestion d'état (Provider)
Les états sont gérés via ~ChangeNotifier~.
- Les variables principales sont actuellement dans ~models/change_notifier.dart~.
- Convention de nommage future : les fichiers prendront le nom de l'entité gérée suivi de ~_notifier.dart~.

*** Refactoring & Migration
Une transition vers une "Clean Architecture" est en cours. L'organisation du code va devenir plus logique au fil des commits, bien que certaines parties du code actuel soient dépréciées ou non utilisées.

** Documentation
La documentation technique est intégrée directement dans le code via des commentaires précédés du tag ~// DOC~.

#+begin_src dart

#+end_src
